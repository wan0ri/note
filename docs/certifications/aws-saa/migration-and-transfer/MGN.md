title: AWS Application Migration Service

## 1. サービス概要

AWS Application Migration Service (AWS MGN) は、オンプレミス環境や他クラウドから AWS へのアプリケーション移行を簡素化・高速化するサービスである。  
物理サーバー、仮想サーバー、クラウドインスタンスを AWS に移行でき、ダウンタイムを最小限に抑えつつ既存アプリケーションを移行できる。  
AWS MGN はサーバーのレプリケーション、カットオーバー、移行後の管理を自動化し、移行プロジェクトの複雑さを軽減する。

主なユースケースは次の通り。

- オンプレミスサーバーの AWS への移行
- 他のクラウド環境からの AWS への移行
- データセンターの縮小・閉鎖、災害復旧戦略
- テスト環境の作成

などが挙げられる。  
AWS MGN は、これらのユースケースに対応する機能と、AWS の他サービスとの統合を提供する。

## 2. 主な特徴と機能

### 2.1 継続的なデータレプリケーション

AWS MGN は、ソース環境のサーバーデータをブロックレベルで AWS に継続的にレプリケーションする。  
これにより、移行中もデータ整合性を維持し、ダウンタイムを最小限に抑えられる。

### 2.2 幅広い OS サポート

Windows Server および Linux の各種 OS をサポートする。  
これにより、様々なアプリケーションを AWS に移行できる。

### 2.3 移行プロセスの自動化

AWS MGN は、サーバーのレプリケーション、カットオーバー、移行後のクリーンアップなどの移行プロセスを自動化する。  
これにより、手動作業を削減し、移行時間を短縮できる。

### 2.4 ターゲット環境のカスタマイズ

移行先となる AWS 環境 (インスタンスタイプ、ストレージ、ネットワーク設定など) をカスタマイズできる。  
これにより、移行後のアプリケーション性能を最適化できる。

### 2.5 テスト移行

本番移行前にテスト移行を実行できる。  
これにより、移行プロセスの検証や移行後の動作確認を安全に行える。

### 2.6 エージェントベースのレプリケーション

ソースサーバーにインストールした AWS Replication Agent を通じてデータをレプリケーションする。  
これにより、既存アプリケーション環境への影響を抑えてデータ移行を行える。

### 2.7 統合性と拡張性

AWS MGN は、Amazon EC2、AWS CloudFormation、AWS Systems Manager などと統合され、AWS への移行を効率的に進められる。  
また、API により移行プロセスの自動化も可能である。

## 3. アーキテクチャおよび技術要素

1. ソースサーバーに AWS Replication Agent をインストール
2. ソースのデータを AWS に継続的にレプリケーション
3. テストカットオーバーまたはカットオーバーを実行し、AWS でインスタンスを起動
4. 必要に応じて移行後の設定やデータ移行を実施
5. 移行完了後、ソースサーバーを停止または削除

AWS Application Migration Service は AWS のインフラ上で提供され、高い可用性とスケーラビリティを備える。  
データレプリケーションや移行は AWS が担うため、利用者はインフラ管理を行う必要がない。

## 4. セキュリティと認証・認可

AWS Application Migration Service は、データ移行のセキュリティ確保のために以下を提供する。

- **IAM 統合**: IAM による AWS MGN へのアクセス制御
- **データ暗号化**: レプリケーションデータは転送中および保存時に暗号化
- **VPC 実行**: ターゲットの EC2 は VPC 内で実行し、ネットワーク隔離を実現
- **アクセス制御**: IAM ポリシーで操作権限を詳細に制御
- **監査ログ**: CloudTrail により利用状況を監査

これらの対策により、移行データを安全に保護し、機密情報を守れる。

## 5. 料金形態

AWS Application Migration Service の料金は主に以下に基づく。

- **レプリケーション時間**: 使用時間に応じて課金
- **データ転送**: 転送量に応じて課金
- **ストレージ**: レプリケーションデータの保存に対する料金
- **その他 AWS サービス**: 移行先インスタンスや EBS など関連サービスの料金

## 6. よくあるアーキテクチャ・設計パターン

AWS Application Migration Service は、様々な移行シナリオで利用できる。  
一般的なパターンは以下の通り。

- **オンプレミスサーバーの AWS への移行**: オンプレミス環境にある物理サーバーや仮想サーバーを AWS EC2 に移行。
- **他のクラウド環境からの AWS への移行**: 他のクラウドプロバイダーの仮想インスタンスを AWS EC2 に移行。
- **データセンターの縮小・閉鎖**: データセンターを縮小または閉鎖するために、サーバーを AWS に移行。
- **災害復旧戦略**: オンプレミスのデータを AWS に継続的にレプリケーションし、災害発生時に AWS でサーバーを起動。
- **テスト環境の作成**: 実稼働環境のデータを AWS にレプリケーションし、テスト環境を作成。

## 7. 設定・デプロイ手順（ハンズオン例）

1. AWS マネジメントコンソールから AWS Application Migration Service を開き、ソースサーバーを登録。
2. ソースサーバーに AWS Replication Agent をインストール。
3. レプリケーションを開始し、データの同期を確認。
4. テストカットオーバーを実行し、AWS 上でインスタンスを起動し、アプリケーションをテスト。
5. 本番カットオーバーを実行し、ソースサーバーを停止し、AWS 上でアプリケーションを起動。

## 8. 試験で問われやすいポイント

### 8.1 継続的なデータレプリケーション

- **機能**: ソース環境のデータを AWS リージョンに継続的にレプリケーション。
- **方法**: ブロックレベルでレプリケーション。
- **試験対策**: データレプリケーションの仕組みやメリットが問われる。

### 8.2 幅広い OS サポート

- **サポート**: Windows と Linux の様々なオペレーティングシステム。
- **目的**: 様々なアプリケーションを AWS に移行可能。
- **試験対策**: 対応 OS の種類や移行シナリオが問われる。

### 8.3 移行プロセスの自動化

- **対象**: サーバーのレプリケーション、カットオーバー、移行後のクリーンアップ。
- **目的**: 手動作業の削減、移行時間の短縮。
- **試験対策**: 自動化されるプロセスやメリットが問われる。

### 8.4 テスト移行

- **目的**: 移行プロセスの検証、移行後のアプリケーション動作確認。
- **利用**: 本番環境への移行前にテストを実施。
- **試験対策**: テスト移行の重要性や利用ケースが問われる。

### 8.5 料金体系

- **課金対象**: レプリケーション時間、データ転送量、ストレージ、移行したインスタンス利用料金。
- **最適化**: 不要なレプリケーション時間の削減や適切なインスタンスタイプ選択でコストを最適化。
- **試験対策**: 料金体系、課金対象が問われる。

### 8.6 類似・関連サービスとの比較

- **AWS SMS**: サーバー移行サービス。MGN は継続レプリケーションを提供
- **AWS CloudEndure Migration**: 旧来の移行サービス。MGN はより簡素化された体験を提供

### 8.7 試験で頻出となる具体的な問われ方と答え

- Q: AWS Application Migration Service (AWS MGN) は何を提供するサービスか。
  - A: オンプレミスや他クラウドから AWS へのアプリケーション移行を簡素化・高速化するサービスである。
- Q: AWS MGN はどのようにデータをレプリケーションするか。
  - A: ブロックレベルで継続的にレプリケーションする。
- Q: AWS MGN はどのような OS をサポートするか。
  - A: Windows Server と Linux の各種 OS をサポートする。
- Q: AWS MGN で自動化される範囲は何か。
  - A: レプリケーション、カットオーバー、移行後のクリーンアップなどである。
- Q: AWS MGN の料金はどのように計算されるか。
  - A: レプリケーション時間、データ転送量、レプリケーションデータのストレージ、移行先インスタンスの利用料金などに基づく。
