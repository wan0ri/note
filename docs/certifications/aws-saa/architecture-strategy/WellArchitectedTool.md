## 1. サービス概要

AWS Well-Architected Tool は、AWS 上で構築されたワークロードを、AWS が推奨するベストプラクティスに基づいてレビューし、改善点を見つけ出すための無料サービス。  
このツールを利用することで、開発者はシステムアーキテクチャの設計、実装、運用において、優れた運用効率、セキュリティ、信頼性、パフォーマンス、コスト効率、持続可能性の 6 つの柱に沿った改善を継続的に行うことができる。  
これにより、システム全体の品質向上、リスク軽減、ビジネス価値の最大化に貢献する。

主なユースケースとして、新規システムの設計レビュー、既存システムの改善、クラウド移行計画の評価、ベストプラクティスの学習などが挙げられる。  
Well-Architected Tool は、開発チームがより高品質なシステムを構築し、運用するためのガイドラインを提供する。

## 2. 主な特徴と機能

### 2.1 6 つの柱

AWS Well-Architected Framework は、以下の 6 つの柱に基づいてアーキテクチャを評価する。

- **優れた運用効率（Operational Excellence）**: システムの運用、監視、改善プロセスを最適化する。自動化、継続的改善、エラーからの学習が重要。
- **セキュリティ（Security）**: 機密性、完全性、可用性を維持するために、セキュリティリスクを特定し、対策を講じる。
- **信頼性（Reliability）**: 障害からの復旧能力、システムの高可用性を確保し、ビジネス要件を満たすための能力を評価する。
- **パフォーマンス効率（Performance Efficiency）**: コンピューティングリソース、ストレージ、データベース、ネットワークなどを適切に活用し、パフォーマンスを最大化する。
- **コスト最適化（Cost Optimization）**: 不必要なコストを削減し、価値を高めるための最適なリソース利用を評価する。
- **持続可能性（Sustainability）**: エネルギー消費量や環境への影響を測定し、リソース使用率の効率性を向上させるための改善プランやアクションを提示する。

### 2.2 ワークロードの定義

Well-Architected Tool では、評価対象となるワークロード（アプリケーション、サービスなど）を定義する。  
ワークロードの定義には、アプリケーションのビジネス上の目的、技術的なアーキテクチャ、運用チームなどの情報を含む。

### 2.3 質問とベストプラクティス

ツール内には、6 つの柱に対応した、数百に及ぶ質問が用意されている。  
これらの質問に回答することで、 ワークロードが AWS のベストプラクティスに沿って設計されているかどうかを評価する。  
質問に沿って、詳細な説明とベストプラクティスの推奨事項が提供される。

### 2.4 改善計画

評価の結果、発見された改善点に基づいて、具体的な改善計画を作成できる。  
ツールは、リスクの高い箇所を特定し、推奨される改善策を提案する。

### 2.5 レビューの共有とコラボレーション

作成したワークロードレビューは、チームメンバーと共有し、コラボレーションしながら改善を進めることができる。  
レビューの結果は CSV 形式でエクスポートしたり、AWS Organizations と統合して組織全体で利用できる。

### 2.6 カスタムレンズ

Well-Architected Tool では、独自のベストプラクティスを定義したカスタムレンズを作成できる。  
これにより、組織の特定のニーズや規制に合わせた評価が可能になる。

### 2.7 統合性と拡張性

AWS Organizations と連携し、組織全体でのワークロードレビューを一元管理できる。  
また、API を通じてツールを自動化することも可能である。

## 3. アーキテクチャおよび技術要素

1. ユーザーは AWS マネジメントコンソールまたは API を通じて Well-Architected Tool にアクセス。
2. ワークロードを定義し、6 つの柱に関する質問に回答。
3. ツールは回答に基づいてリスクを特定し、改善計画を提示。
4. ユーザーは改善計画に基づいてワークロードを修正し、必要に応じて再レビューを実施。
5. レビュー結果はチーム内で共有し、継続的な改善を行う。

AWS Well-Architected Tool はフルマネージドサービスであり、サーバーレスアーキテクチャで構成されている。  
可用性、スケーラビリティは AWS が管理し、ユーザーはインフラの管理を行う必要はない。

## 4. セキュリティと認証・認可

AWS Well-Architected Tool のセキュリティは、AWS の標準的なセキュリティプラクティスに準拠している:

- **IAM 統合**: AWS Identity and Access Management (IAM) を利用して、ツールへのアクセスを制御する。
- **暗号化**: 保存データと転送中のデータを暗号化する。
- **アクセス制御**: IAM ポリシーを通じて、ユーザーやグループごとに、ツールへのアクセス権限を詳細に制御できる。
- **監査ログ**: CloudTrail を通じて、ツールの利用状況を監査できる。

これにより、ツールの利用に対する不正アクセスを防止し、機密情報を保護できる。

## 5. 料金形態

AWS Well-Architected Tool は、**無料**で利用できるサービス。

ただし、ツールで推奨される改善策を実行するために、他の AWS サービス（EC2、S3、Lambda など）を利用する際には、それらのサービスの料金が発生する。

## 6. よくあるアーキテクチャ・設計パターン

Well-Architected Tool は、様々なワークロードに対して利用できる。  
一般的なパターンは以下の通り:

- **新規システム設計**: 新規の AWS 環境を構築する前に、アーキテクチャ設計を評価し、ベストプラクティスを適用する。
- **既存システム改善**: 既存の AWS 環境を定期的にレビューし、改善点を洗い出す。
- **クラウド移行**: オンプレミス環境から AWS への移行計画を評価し、最適な移行戦略を策定する。
- **セキュリティ強化**: セキュリティの脆弱性を特定し、対策を講じる。
- **コスト最適化**: リソースの利用状況を分析し、コスト削減の機会を見つける。
- **継続的な改善**: システムの運用状況を継続的に監視し、改善を繰り返す。

## 7. 設定・デプロイ手順（ハンズオン例）

1. AWS マネジメントコンソールから AWS Well-Architected Tool を開く。
2. 新しいワークロードを作成し、ワークロードに関する基本情報を入力。
3. 6 つの柱に関する質問に回答し、評価を実施。
4. ツールが提示するリスクと改善計画を確認。
5. 改善計画に基づいて、システム構成を修正。
6. 必要に応じて、再度レビューを実施し、継続的な改善を行う。

- **重要性**: 標準的な評価に加え、より具体的な評価ができる。

### 15.6 料金体系

- **料金**: Well-Architected Tool 自体は無料。
- **関連料金**: ツールで推奨される改善策を実行する際には、関連する AWS サービスの料金が発生。
- **注意点**: ツールだけでなく、関連サービスの料金も考慮する必要がある。

### 15.7 類似・関連サービスとの比較

- **AWS Trusted Advisor**: コスト削減、セキュリティ、パフォーマンスの改善に関する推奨事項を提供するサービス。Well-Architected Tool は、より包括的なアーキテクチャ評価を行う。
- **AWS Cost Explorer**: クラウド利用コストの分析・可視化に特化したサービス。Well-Architected Tool はアーキテクチャの設計と改善に焦点を当てる。
- **AWS Security Hub**: セキュリティのベストプラクティスに従って、セキュリティの状態を評価するサービス。Well-Architected Tool はセキュリティを含む 6 つの柱で評価を行う。

### 15.8 試験で頻出となる具体的な問われ方と答え

- Q: AWS Well-Architected Tool の 6 つの柱は何？
  - A: 優れた運用効率、セキュリティ、信頼性、パフォーマンス効率、コスト最適化、持続可能性。
- Q: Well-Architected Tool は何に使用するか？
  - A: AWS 環境のアーキテクチャをベストプラクティスに基づき評価し、改善点を特定するために使用する。
- Q: ワークロードの定義とは何？
  - A: 評価対象となるアプリケーション、サービスのビジネス上の目的、技術的なアーキテクチャ、運用チームなどの情報をまとめたものである。
- Q: Well-Architected Tool の利用料金は？
  - A: 無料で利用可能。
- Q: カスタムレンズはどのような目的で使用するか？
  - A: 組織独自のベストプラクティスを定義し、より具体的な評価を行うために使用する。
